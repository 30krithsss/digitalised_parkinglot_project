                          REST API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════

SLOTS API:
  GET    /api/slots
         Get all parking slots

  GET    /api/slots/:slotNumber
         Get specific slot details

  PUT    /api/slots/:slotNumber
         Update slot status (OCR detection, manual entry)

  POST   /api/slots/seed/all
         Initialize 100 demo slots

AUTHENTICATION API:
  POST   /api/auth/register
         User registration
         Body: { email, password }

  POST   /api/auth/login
         User login
         Returns: JWT token

  GET    /api/auth/profile
         Get authenticated user profile
         Header: Authorization: Bearer <token>

  POST   /api/auth/logout
         Logout & invalidate token

PAYMENTS API:
  POST   /api/payments/bill
         Calculate parking bill
         Body: { slotNumber, entryTime, exitTime }

  GET    /api/payments/:id
         Get payment record

  POST   /api/payments/qr
         Generate UPI QR code

═══════════════════════════════════════════════════════════════════════════════
                         SOCKET.IO REAL-TIME EVENTS
═══════════════════════════════════════════════════════════════════════════════

SERVER EVENTS (Broadcast to all clients):

slot-update
  When: Slot status changes
  Data: { slotNumber, status, licensePlate, timestamp }
  Effect: Dashboard updates in real-time

occupancy-change
  When: Overall occupancy percentage changes
  Data: { totalOccupied, totalAvailable, percentage }
  Effect: Admin panel & kiosk refresh

payment-complete
  When: Payment is processed
  Data: { userId, slotNumber, amount, timestamp }
  Effect: Notification to user

admin-alert
  When: Admin performs action
  Data: { action, detail, timestamp }
  Effect: System log update

═══════════════════════════════════════════════════════════════════════════════
                       SYSTEM FLOW EXAMPLES
═══════════════════════════════════════════════════════════════════════════════

SCENARIO 1: CAR ENTRY

1. Car arrives at parking lot
2. AI camera captures license plate
3. Tesseract.js processes image → Recognizes plate
4. Backend receives plate data
5. System finds nearest available slot (e.g., A-05)
6. Updates database: Slot A-05 status = "occupied"
7. Triggers smart lock to open gate
8. Socket.io broadcasts update to all clients
9. 3D Dashboard shows slot A-05 as red (occupied)
10. Admin panel updates occupancy percentage
11. Email confirmation sent to user
12. Display kiosk updates available slot count

SCENARIO 2: PAYMENT PROCESSING

1. User enters parking duration in payment app
2. System checks entry and exit times
3. Calculates pricing:
   • If peak hours: ₹20 × 1.5
   • If off-peak: ₹20 × 1.2
4. Generates total bill amount
5. Creates UPI QR code
6. User scans and pays
7. Payment gateway confirms
8. Database records transaction
9. Slot status changes to "vacant"
10. Smart lock opens for exit
11. User receives receipt email
12. Dashboard updates occupancy

                      TROUBLESHOOTING GUIDE
═══════════════════════════════════════════════════════════════════════════════

MONGODB NOT CONNECTING:
  ✗ Error: ECONNREFUSED 127.0.0.1:27017
  ✓ Fix: Start MongoDB with 'mongod' command
  ✓ OR: Use MongoDB Atlas (update .env with connection string)

PORT ALREADY IN USE:
  ✗ Error: listen EADDRINUSE :::5000
  ✓ Fix: netstat -ano | findstr :5000
  ✓ Then: taskkill /PID <process_id> /F

FRONTEND WON'T CONNECT:
  ✗ Error: CORS error or "Cannot reach backend"
  ✓ Fix: Ensure backend is running on localhost:5000
  ✓ Check: Backend has CORS enabled

3D DASHBOARD NOT RENDERING:
  ✗ Error: "3D visualization not displaying"
  ✓ Fix: Clear browser cache (Ctrl+Shift+Delete)
  ✓ Try: Different browser (Chrome recommended)
  ✓ Update: Graphics drivers

DEPENDENCIES MISSING:
  ✗ Error: "Cannot find module"
  ✓ Fix: Run 'npm install' in both backend/ and frontend/
