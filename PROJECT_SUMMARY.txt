🗄️ MONGODB SETUP
─────────────────

Database Name: iot-parking-lot

Collections:
  1. slots (100 slots across 10 zones)
     {
       slotNumber: "A-01",
       zone: "A",
       status: "available" | "occupied" | "reserved",
       licensePlate: "KA-51-AB-1234" (if occupied),
       entryTime, exitTime, timestamps
     }

  2. users (authentication)
     { email, password (hashed), role: "user" | "admin", subscription }

  3. payments (billing records)
     { userId, slotNumber, amount, duration, peakMultiplier, status }

🚀 HOW TO RUN
──────────────

OPTION 1: Using Batch Files (Windows)

  Terminal 1:
  > run_backend.bat
  
  Terminal 2:
  > run_frontend.bat
  
  Terminal 3 (after both are running):
  > seed_database.bat

OPTION 2: Using Command Prompt

  Terminal 1 (Backend):
  > cd backend
  > npm run dev
  
  Terminal 2 (Frontend):
  > cd frontend
  > npm start
  
  Terminal 3 (Seed DB):
  > curl -X POST http://localhost:5000/api/slots/seed/all

OPTION 3: Quick System Check

  > check_system.bat
  (Verifies Node.js, npm, MongoDB, and dependencies)

🔌 MAIN API ENDPOINTS
──────────────────────

Slots:
  GET    /api/slots                     - Get all slots
  GET    /api/slots/:slotNumber         - Get specific slot
  PUT    /api/slots/:slotNumber         - Update slot status
  POST   /api/slots/seed/all            - Initialize 100 slots

Authentication:
  POST   /api/auth/register             - Create user account
  POST   /api/auth/login                - Login (returns JWT)
  GET    /api/auth/profile              - Get user profile
  POST   /api/auth/logout               - Logout

Payments:
  POST   /api/payments/bill             - Calculate parking bill
  GET    /api/payments/:id              - Get payment record

📊 DATA FLOW EXAMPLE: CAR ENTRY
───────────────────────────────

1. Car arrives → IoT camera/sensor detects
2. License plate recognized → Tesseract.js OCR
3. Backend processes → Verify & find slot
4. Database updates → Slot marked as occupied
5. Smart lock activates → GPIO relay controlled
6. Real-time broadcast → Socket.io notifies all clients
7. 3D Dashboard updates → Slot changes color (red)
8. Admin panel refreshes → Shows new occupancy
9. Confirmation sent → Email/SMS to user

❓ TROUBLESHOOTING
────────────────────

Problem: MongoDB connection fails
→ Solution: Start mongod OR use MongoDB Atlas connection string in .env

Problem: Port 5000 or 3000 already in use
→ Solution: Check with netstat and kill process, or change port in .env

Problem: Frontend won't connect to backend
→ Solution: Ensure backend is running (localhost:5000), check CORS in server.js

Problem: 3D Dashboard not rendering
→ Solution: Update graphics drivers, try different browser, clear cache

Problem: Dependencies missing
→ Solution: Run npm install in both backend/ and frontend/ folders
